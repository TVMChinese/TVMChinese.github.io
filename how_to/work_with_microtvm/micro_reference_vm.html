microTVM 虚拟机¶

作者: Andrew Reusch

此教学手册教您如何启动microTVM虚拟机。通过此手册，您可以在不安装microTVM依赖库的情况下使用物理设备进行开发。同时，当您需要使用microTVM重现一些代码行为时，例如编写bug报告，此手册能够提供有效的帮助。

microTVM可以有效地使用TVM在低端微控制器上构建和运行模型。microTVM旨在与各种各样的SoC和运行时环境（如裸机、RTOS等）相互兼容。然而，开发人员需要一些必要的软件依赖环境才能够共享和再现bug和运行结果。microTVM虚拟机旨在提供这些环境。

工作原理¶

TVM代码仓库中并没有提供虚拟机，在 apps/microtvm/reference-vm 目录下的文件描述了如何使用 Vagrant 虚拟机构建工具来创建一个microTVM虚拟机。

MicroTVM虚拟机分为两个部分：

  1. 一个Vagrant基础镜像，它包含了所有必要的软件依赖环境，构建该镜像的脚本保存在 apps/microtvm/reference-vm/<platfrom>/base-box 。当平台的“稳定”依赖环境发生变化时，TVM提交者将会运行这个脚本来生成新的基础镜像，该镜像会被上传到 Vagrant Cloud 。

  2. 一个虚拟机工作环境，用户通常使用基础镜像作为起点来构建它。构建脚本保存在 apps/microtvm/reference-vm/<paltform> （除了 base-box ）。

设置虚拟机¶

安装必要软件¶

这些必要软件包括：

  1. Vagrant

  2. 一个虚拟机管理程序（ VirtualBox 、 Parallels 或 VMWare Fusion/Workstation ）。TVM推荐使用 VirtualBox ，因为它是免费的虚拟机管理程序，但请注意， VirtualBox Extension Pack 是使用USB转发功能所必要的扩展包。如果选择使用VirtualBox，您还可以考虑安装 vbguest 插件。

  1. 如果您还需要其他的插件， Vagrant provider plugin 将提供更多程序插件（或在 此处 查找有关VMWare的插件）。

首次启动¶

当您第一次使用microTVM虚拟机时，您需要在本地创建一个镜像，然后给它提供必要准备。

# Replace zephyr with the name of a different platform, if you are not using Zephyr.
~/.../tvm $ cd apps/microtvm/reference-vm/zephyr
# Replace <provider_name> with the name of the hypervisor you wish to use (i.e. virtualbox, parallels, vmware_desktop).
~/.../tvm/apps/microtvm/reference-vm/zephyr $ vagrant up --provider=<provider_name>

这段命令行需要一些时间来运行，同时，它将在您的机器上占用4到5GB的存储空间。这些代命令行的作用如下：

  1. 下载 MicroTVM基础镜像 ，然后使用它生成一个特定于这个TVM目录的新虚拟机。

  2. 将TVM目录挂载到虚拟机（如果使用 git-subtree ， .git 根目录将会被挂载）。

  3. 构建TVM并安装与之相对应的Python虚拟环境。

将硬件连接到虚拟机¶

接下来，您需要配置 USB 转发设置，以便通过USB接口将物理开发板连接到虚拟机（而不是直接连接到主机上的操作系统）。

建议您设置设备筛选器，而不是设置单次转发，因为在编程过程中设备通常会重新启动，而此时您需要再次启用转发。当这种情况发生时，开发用户可能不易察觉需要再次启动转发。开启设备筛选器的方法如下：

  • VirtualBox

  • Parallels

  • VMWare Workstation

在microTVM虚拟机中重新构建TVM¶

在第一次启动之后，每当您修改了C++运行时代码或者切换到不同版本的时候，您都需要重新构建TVM以保持构建文件是最新的，这些构建文件在 $TVM_HOME/build-microtvm-zephyr 目录下。您可以通过重新配置虚拟机（在之前运行 vagrant up 命令的目录下运行 vagrant provision ），又或者自己手动重新构建TVM。

请注意：在microTVM虚拟机中的TVM构建文件 .so 与您主机上的构建文件是不同的。这也是为什么要将构建文件放置在 build-microtvm-zephyr 这个特殊目录下。

登录虚拟机¶

虚拟机应该向主机提供唯一的主机名 microtvm 。您可以通过SSH登录到虚拟机：

$ vagrant ssh

然后 cd 到主机上访问TVM的相同路径。例如，在 Mac 上：

$ cd /Users/yourusername/path/to/tvm

测试¶

当microTVM虚拟机完成配置工作，您可以使用 poetry 运行测试用例：

$ cd apps/microtvm/reference-vm/zephyr
$ poetry run python3 ../../../../tests/micro/qemu/test_zephyr.py --zephyr-board=stm32f746g_disco

如果您没有硬件设备，但仍希望运行测试用例，您可以在microTVM虚拟机上运行本地QEMU模拟器，详细命令如下：

$ cd /Users/yourusername/path/to/tvm
$ cd apps/microtvm/reference-vm/zephyr/
$ poetry run pytest ../../../../tests/micro/qemu/test_zephyr.py --zephyr-board=qemu_x86

Docker 镜像¶

我们提供 docker 的实用程序脚本，以帮助开发人员设置开发环境。它们还有助于通过 TVM 演示和教程运行。如果我们想使用 cuda， 我们需要 docker 和 nvidia-docker

获取 tvm 源代码分发或克隆 github 存储库以获取辅助脚本

git clone --recursive https://github.com/apache/tvm tvm

然后，我们可以使用以下命令来启动 docker 镜像。

/path/to/tvm/docker/bash.sh <image-name>

这里的 image-name 可以是本地镜像的镜像名称，例如，在完成本地构建后 “tvm.ci_cpu”。

此辅助脚本可进行以下工作：

  • 将当前目录挂载到 /workspace

  • 将用户切换为调用 bash.sh 的同一用户（这样您就可以读/写主机系统）

  • 在 Linux 上使用主机端网络。 使用桥接网络并在 macOS 上公开端口 8888，因为不支持主机网络驱动程序。 （所以你可以使用 jupyter notebook）

然后你可以通过输入 `jupyter notebook 来启动一个 jupyter notebook

jupyter notebook

在 macOS 上启动 jupyter 笔记本时，您可能会看到错误 OSError: [Errno 99] Cannot assign requested address 。 您可以通过以下方式更改绑定 IP 地址

jupyter notebook --ip=0.0.0.0

请注意，在 macOS 上，因为我们使用桥接网络，据报道 jupyter notebook 将在类似 http://{container_hostname}:8888/?token=... 的 URL 上运行。 将 container_hostname 粘贴到浏览器时，应将其替换为 localhost。

Docker 源¶

如果您有兴趣构建自己的 docker 镜像，请查看 The docker source 。

运行以下命令来构建 docker 镜像。

/path/to/tvm/docker/build.sh <image-name>

您可以找到一些非官方构建的第三方镜像。 这些镜像用于测试目的，不属于 ASF 版本。

https://hub.docker.com/r/tlcpack/.
